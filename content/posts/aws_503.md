## AWS の 503 とは何か？（公式の定義）

**503** は、[HTTP](/glossary/http/)標準仕様（[RFC](/glossary/rfc/) 9110）で定められている[ステータスコード](/glossary/ステータスコード/)の一つです。

HTTP 503 Service Unavailable は、サービスが一時的に利用できない状態を示します。AWS の文脈では、AWS のサービス側に問題が生じており、リクエスト（サーバーへの処理依頼）を処理できないことを意味します。この状態は通常、一時的なものであり、時間をおくことで解決することが多いです。

---

## このエラーが発生する主な原因

AWS で 503 が出るときに、最もよく見られる原因を挙げます。

- サーバーやリソースの処理能力が上限に達している
- AWS サービス側で障害やメンテナンスが発生している
- バックエンドサーバー（実際に処理を行うサーバー）が応答しない、または停止している
- 一時的なネットワークの接続問題
- 設定ミスによるリソース割り当てエラー

---

## 具体的な解決手順

上の原因ごとの対処法を、実行できる手順の形でまとめました。上から順番に試してください。

1. **AWS Service Health ダッシュボード**で現在の障害情報を確認する
2. **Auto Scaling**（自動的にサーバー数を調整する機能）の設定を見直し、必要な処理能力を確保する
3. **CloudWatch ログ**（AWS のログ確認ツール）を確認し、バックエンドでエラーが起きていないか調べる
4. 数分待ってからリクエストを再度試みる

それでも解決しない場合は、次の方法を試してください。

- AWS の公式ドキュメントサイトで最新情報を確認する
- エラーメッセージの全文をコピーして検索エンジンで調べる
- AWS の公式サポートコミュニティに質問する、または有償サポートに問い合わせる
