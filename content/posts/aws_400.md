## AWS の 400 とは何か？（公式の定義）

**400** は、[HTTP](/glossary/http/)標準仕様（[RFC](/glossary/rfc/) 9110）で定められている[ステータスコード](/glossary/ステータスコード/)の一つです。

[HTTP](/glossary/http/) 400 は「Bad Request（不正な[リクエスト](/glossary/リクエスト/)）」を意味し、クライアント側から送信された[リクエスト](/glossary/リクエスト/)に構文的な誤りがある場合に返されます。AWS の文脈では、AWSサービスへの[リクエスト](/glossary/リクエスト/)に[パラメータ](/glossary/パラメータ/)（設定値）の誤りがある場合に表示されます。

---

## このエラーが発生する主な原因

AWS で 400 が出るとき、よく見られる原因を以下に示します。

- 必須の[パラメータ](/glossary/パラメータ/)が不足しているか、データ型（数値か文字列かなど）が間違っている
- ARN（Amazon Resource Name、AWSリソースを一意に特定するための識別子）の形式が誤っている
- 使用している[SDK](/glossary/sdk/)（AWS と連携するためのプログラムツール）や[CLI](/glossary/cli/)（コマンドライン操作ツール）のバージョンが古い
- リクエストの本体（JSON や XML）に不正な形式が含まれている

---

## 具体的な解決手順

上記の原因ごとの対処法を、実行順に示します。上から順に試してください。

1. AWSの[エラーレスポンス](/glossary/エラーレスポンス/)（エラー応答メッセージ）内の「message」フィールドで不足[パラメータ](/glossary/パラメータ/)や誤りを特定します
2. ARN を AWS の管理[コンソール](/glossary/コンソール/)（ウェブ画面）からコピー&ペーストして使用します
3. 使用している[SDK](/glossary/sdk/)や[CLI](/glossary/cli/)を最新版に更新します

それでも解決しない場合は、以下の方法を試してください。

- AWS の公式ドキュメントで最新の仕様情報を確認する
- エラーメッセージの全文をコピーして検索エンジンで調べる
- 公式のコミュニティフォーラムや AWS サポートに問い合わせる
