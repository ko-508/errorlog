## AWS の 404 とは何か？（公式の定義）

**404** は、[HTTP](/glossary/http/)標準仕様（[RFC](/glossary/rfc/) 9110）で定められている[ステータスコード](/glossary/ステータスコード/)の一つです。

AWS の文脈では、このコードは次のことを意味します。

> 指定した AWS リソース（S3 バケット・EC2 インスタンスなど）が見つかりません。

このエラーが表示されたときは、次の「原因」の節を確認してください。多くの場合、設定の見直しや手順の確認だけで解決できます。

---

## このエラーが発生する主な原因

AWS で 404 が出るときに、最もよく見られる原因を挙げます。自分の状況に当てはまるものを探してください。

- リソース名または ID のスペルが間違っている
- リソースが別のリージョン（AWS のデータセンター地域）に存在している
- リソースがすでに削除されている
- リソースへのアクセス権限（IAM ポリシー）が不足している
- API エンドポイント（外部サービスへのアクセス口）の URL が誤っている

---

## 具体的な解決手順とチェックリスト

上の原因ごとの対処法を、実行できる手順の形でまとめました。上から順番に試すことで、多くの場合は解決に向かいます。

1. AWS マネジメントコンソール（管理画面）で対象リソースの存在とリージョンを確認する
1. AWS [CLI](/glossary/cli/)（コマンドラインツール）で `--region` パラメーター（設定値）を明示的に指定してコマンドを実行する
1. リソースの命名規則（大文字小文字の区別、ハイフンなど）を再確認する
1. リソースのアクセス権限設定を確認し、必要に応じて IAM ポリシーを更新する
