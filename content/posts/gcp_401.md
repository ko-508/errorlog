## GCP の 401 エラーが出た時の対処法

GCP（Google Cloud Platform）で 401 エラーが発生するのは、Google Cloud [API](/glossary/api/) への[認証](/glossary/認証/)情報（[認証](/glossary/認証/)用の資格情報）が無効になっているか、有効期限が切れている状態です。このエラーが出ると、Cloud Functions やデータベースなど GCP のサービスが一切利用できなくなります。

## よくある原因

**1. [サービスアカウント](/glossary/サービスアカウント/)キーの期限切れ**

[サービスアカウント](/glossary/サービスアカウント/)キー（[JSON](/glossary/json/) ファイル形式の[認証](/glossary/認証/)情報）には有効期限があります。長期間使用していなかったり、セキュリティ[ポリシー](/glossary/ポリシー/)で期限切れになっている場合があります。特に本番環境では定期的なキーローテーションが行われるため、古いキーを使い続けると 401 エラーが発生します。

**2. Application Default Credentials（ADC）の未設定**

ADC とは、GCP が自動的に[認証](/glossary/認証/)情報を探す仕組みです。[環境変数](/glossary/環境変数/)が正しく設定されていない、または gcloud コマンドでのログイン情報が失われている場合、[API](/glossary/api/) 呼び出しが[認証](/glossary/認証/)できず 401 エラーになります。

**3. アクセス[トークン](/glossary/トークン/)の有効期限切れ**

[API](/glossary/api/) 呼び出しに使うアクセス[トークン](/glossary/トークン/)は 1 時間で自動的に無効になります。長時間連続実行するプログラムで[トークン](/glossary/トークン/)をリフレッシュ（更新）していないと、1 時間経過後に 401 エラーが発生します。

## 解決手順

**ステップ 1: アクティブなアカウントを確認する**

まず、現在どのアカウントで gcloud が設定されているか確認しましょう。

```bash
gcloud auth list
```

出力結果から、使いたいアカウントの横に `*` マークがついているか確認してください。

**ステップ 2: Application Default Credentials を再設定する**

ADC をリセットして再ログインします。

```bash
gcloud auth application-default login
```

このコマンドを実行するとブラウザが自動的に開き、Google アカウントでのログインが求められます。ログインが完了すると、[認証](/glossary/認証/)情報がローカルに保存されます。

**ステップ 3: [サービスアカウント](/glossary/サービスアカウント/)キーを再生成する**

[サービスアカウント](/glossary/サービスアカウント/)を使用している場合、Google Cloud Console で新しいキーを生成します。

1. Google Cloud Console を開き、メニューから **「[IAM](/glossary/iam/) と管理」→「[サービスアカウント](/glossary/サービスアカウント/)」** に移動
2. 対象の[サービスアカウント](/glossary/サービスアカウント/)をクリック
3. **「キーを追加」→「新しいキーを作成」** を選択
4. ファイル形式は **[JSON](/glossary/json/)** を選択し、ダウンロード
5. [環境変数](/glossary/環境変数/) `GOOGLE_APPLICATION_CREDENTIALS` に新しいキーのパスを設定

```bash
export GOOGLE_APPLICATION_CREDENTIALS="/path/to/new-key.json"
```

## それでも解決しない場合

- [キャッシュ](/glossary/キャッシュ/)された古い[認証](/glossary/認証/)情報が残っていることもあります。`~/.config/gcloud/` ディレクトリを確認し、古いセッション情報を削除してください
- [IAM](/glossary/iam/)（Identity and Access Management）で[サービスアカウント](/glossary/サービスアカウント/)に必要な権限が付与されているか確認してください
- ファイアウォール設定や[ネットワーク](/glossary/ネットワーク/)の問題で GCP [API](/glossary/api/) に接続できないケースもあるため、通信状況も確認しましょう
