> この記事では、**GitHub [API](/glossary/api/)** を使っているときに表示される **403** というエラーの意味と、その直し方を順を追って説明します。

---

## GitHub API の 403 とは何か？（公式の定義）

**403** は、[HTTP](/glossary/http/)標準仕様（[RFC](/glossary/rfc/) 9110）で定められている[ステータスコード](/glossary/ステータスコード/)の一つです。

GitHub [API](/glossary/api/) の文脈では、このコードは次のことを意味します。

> [認証](/glossary/認証/)は成功したが、その操作の権限がない。

このエラーが出たときは、慌てずに次の「原因」の節を確認してください。
多くの場合、設定の見直しや手順の確認だけで解決できます。

---

## このエラーが発生する主な原因（起きる理由の整理）

GitHub [API](/glossary/api/) で 403 が出るときに、最もよく見られる原因を挙げます。
自分の状況に当てはまるものを探してみてください。

- 使用している[トークン](/glossary/トークン/)に必要な[スコープ](/glossary/スコープ/)（権限）が付与されていない
- 組織の管理者が[API](/glossary/api/)アクセスを制限するSSO設定をしている
- GitHubの標準的な[レート制限](/glossary/レート制限/)に達している

---

## 具体的な解決手順とチェックリスト（順番どおりに試す）

上の原因ごとの対処法を、実行できる手順の形でまとめました。
**上から順番に試す**ことで、多くの場合は解決に近づけます。

1. [トークン](/glossary/トークン/)の設定画面で[スコープ](/glossary/スコープ/)一覧を確認し、必要なものを追加する
1. Organization設定でSSOの[認可](/glossary/認可/)を行う
1. X-RateLimit-Remaining レスポンスヘッダーで残り[リクエスト](/glossary/リクエスト/)数を確認する

---

## まとめ

GitHub [API](/glossary/api/) の **403** エラーは、上記のいずれかの原因によって発生するケースがほとんどです。
チェックリストを一つずつ確認することで、大半の問題は自力で解決できます。

それでも解決しない場合は、次の方法を試してください。

- GitHub [API](/glossary/api/) の公式ドキュメントで最新の情報を確認する
- エラーメッセージの全文をコピーして検索エンジンで調べる
- 公式のコミュニティフォーラムやサポートに問い合わせる
